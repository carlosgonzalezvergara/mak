1. Overview

M.A.K. is a browser-based diagramming tool designed for syntactic analysis within the framework of Systemic Functional Linguistics (SFL). It allows users to build constituency tree diagrams interactively, assigning multiple rows of labels per node to reflect the simultaneous functional layers characteristic of SFL analysis.

M.A.K. is a manual assistant: the user makes all analytical decisions (choosing where to divide constituents, what labels to assign, and how to organise the tree) while the program handles hierarchical visualisation, label management, and image export.

The tool runs entirely in the browser. No data is uploaded to any server; all work is stored locally in the browser or saved as downloadable files.

2. The interface

M.A.K. uses a split layout with two areas:

  • Left panel — a narrow sidebar containing the logo, the ✦ New Diagram button, file operations (Save, Load, Export), history controls (Undo, Redo), zoom controls, and a collapsible quick-reference guide.
  • Canvas — the main area where the tree diagram is displayed and edited. When no diagram is active, the canvas shows a placeholder message.

Zoom

The zoom controls are located in the left panel under the Zoom section. Use [−] and [+] to adjust the view, or click the percentage indicator to reset to 100%. The zoom range is 40%–200% in 20% increments.

Auto-save

M.A.K. automatically saves your work to the browser's local storage every few seconds. If you close the tab or browser and return later, M.A.K. will offer to restore your last session. This auto-save is stored locally only — nothing is uploaded.

Note: Auto-saved data may be lost if you clear your browser's site data. For permanent storage, use the Save function to download a .mak file.

3. Creating a new diagram

  1. Click ✦ New Diagram in the left panel.
  2. A dialog box appears. Type the expression you want to analyse in the text field.
  3. If you need IPA characters, click the t͡ʃ IPA button to open the IPA character picker. Characters are inserted at the cursor position in the text field.
  4. Click Analyse (or press Enter). The root node is created with the full expression.

From that point, your work consists of dividing constituents, assigning labels, and — when necessary — adding extra nodes or editing node text.

The IPA character picker

The picker is organised into categories: vowels, plosives, nasals, fricatives, approximants, affricates, clicks, suprasegmentals, diacritics (voicing, place, manner, secondary), tones, and bracketing symbols. Click any character to insert it into the expression field. The picker closes when you dismiss the dialog or toggle it off.

Undo and redo

M.A.K. maintains a history of up to 50 states. Use the Undo and Redo buttons in the left panel, or the keyboard shortcuts (see section 10), to step back and forth through your editing history.

4. Division mechanics

Dividing a node

To divide a constituent into smaller parts, click on a tinted node box (the node must contain more than one word). M.A.K. enters division mode and displays the words with clickable cut bars between them.

  • Click one or more grey bars between words to mark the boundaries between constituents. Selected bars turn dark.
  • Click Divide to generate the child nodes.
  • Click Cancel to exit division mode without making changes.

Consolidating a node

If a node contains more than one orthographic word but should be treated as a single syntactic unit (e.g. a phrasal verb or a proper name), you can consolidate it to prevent division. Hover over the node and click the ∪ button in the upper-left corner. The node will no longer be divisible. To reverse this, click the ÷ button that replaces it.

Undoing a division

If a node has been divided into children and you want to reverse that, hover over it and click the ↩ button on the right side of the node box. This removes all children and restores the node to its undivided state.

Clearing node text

You can remove a node's text while keeping its labels and children intact. Hover over the node and click the ✕ button in the upper-right corner. This is available on every node, including the root.

5. The label system

M.A.K.'s label system is designed for SFL analysis, where a single constituent may need to be labelled simultaneously in several functional layers (e.g. transitivity, mood, theme).

Label placement

  • The root node displays its labels below the text box.
  • All other nodes display their labels above the text box.

Multiple label rows

Each node supports between 1 and 5 rows of labels. Non-root nodes are created with 2 rows by default; the root node starts with 1.

  • Hover over a label stack and click the + button to add a new row (up to 5).
  • Click the × button next to any row to remove it (minimum 1 row).

Editing labels

Click on any label to enter edit mode. Type the desired value and press Enter or click elsewhere to confirm. You can use Tab to advance to the next label and Shift+Tab to go back.

Formatting within labels

Labels support inline text formatting using simple markup:

MarkupResultExample
**text**Bold**Actor**
*text*Italic*Theme*
_text_SubscriptParticipant_1_
__text__SuperscriptNP__2__

6. Extra nodes and elision

Adding an extra child node

Once a node has been divided into children, you can insert additional child nodes. This is useful for representing elided or implicit constituents. Hover over the parent node and click the + button at the bottom of the node box.

  1. A prompt will ask for the text of the new node (this can be left empty).
  2. A position picker dialog appears, letting you choose where to insert the new node: At the beginning, At the end, or Between "X" and "Y".

Extra nodes are marked internally as inserted nodes and behave like any other node — they can receive labels and be further divided.

7. Editing node text

M.A.K. allows you to edit the text of any node directly in the tree, without needing to undo and redo divisions. Hover over a node and click the ✎ button in the bottom-right corner.

The node's text becomes an editable input field. Type the new text and press Enter or click outside to confirm. This is useful for correcting typos, adjusting word boundaries, or modifying individual words after the initial division.

8. File management

M.A.K. uses files with the .mak extension to save and load diagrams. These are JSON files that preserve the entire tree structure, all labels, and the original expression.

Save

Click Save in the left panel (or press Ctrl+S / ⌘+S). The file is downloaded to your computer. The filename is generated automatically from the first words of the expression.

Load

Click Load in the left panel and select a .mak file from your computer. M.A.K. also accepts .ent files from the ENT tool; these are automatically migrated to M.A.K.'s format upon loading.

Compatibility note: When loading an .ent file, M.A.K. converts the two-label system (function + category) used by ENT into M.A.K.'s multi-row label system. The upper label (labelTop) becomes the first label row. The lower label (labelBottom), if present, is preserved where applicable.

9. Exporting as PNG

M.A.K. exports the diagram as a PNG image with a transparent background, suitable for insertion into documents, slides, or academic papers.

Click Export as PNG in the left panel. During export, M.A.K. automatically:

  • Resets the zoom to 100% for consistent output.
  • Hides all interactive buttons (edit, delete, add, undo) so they do not appear in the image.
  • Hides empty label rows to produce clean, uninterrupted connector lines.
  • Converts label inputs into static text elements for accurate rendering.

After the image is captured, all interactive elements are restored. The exported image is downloaded automatically.
